# Supabase Checkout Setup

This project now includes a checkout form at `/checkout` that saves rows into Supabase table `checkout_information`.

## 1) Create the table in Supabase

Open SQL Editor in Supabase and run:

```sql
create table if not exists public.checkout_information (
  id bigint generated by default as identity primary key,
  full_name text not null,
  phone text not null,
  email text,
  location text,
  service text not null,
  item_name text not null,
  quantity integer not null check (quantity > 0),
  notes text,
  created_at timestamptz not null default now()
);

alter table public.checkout_information enable row level security;

drop policy if exists "Allow anon insert checkout info" on public.checkout_information;
create policy "Allow anon insert checkout info"
on public.checkout_information
for insert
to anon
with check (true);
```

## 2) Add environment variables locally

Copy `.env.local.example` to `.env.local` and put your real values:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

## 3) Add GitHub Secrets for production deploy

In GitHub repo settings, create these secrets:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`

The workflow `.github/workflows/deploy.yml` now injects those values at build time.

## 4) Test

1. Run locally: `npm run dev`
2. Open `/checkout`
3. Submit sample data
4. Verify row appears in Supabase table `checkout_information`
